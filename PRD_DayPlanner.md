# PRD — Планировщик распорядка дня (Day Planner)

**Версия:** 1.0  
**Дата:** май 2026  
**Статус:** Draft

---

## 1. Обзор продукта

### 1.1 Цель

Создать персональный веб-сайт-планировщик, который помогает пользователю структурировать ежедневный распорядок: отслеживать спортивные активности, здоровье и рабочие задачи в одном месте.

### 1.2 Проблема

Пользователю сложно держать в голове или разрозненно хранить:
- расписание спортивных тренировок,
- приём таблеток, записи к врачам, косметологические процедуры,
- рабочие встречи и совещания.

### 1.3 Решение

Дружелюбный веб-планировщик с тремя тематическими блоками (Спорт / Здоровье / Работа), таймлайном дня и системой добавления/редактирования событий.

### 1.4 Целевая аудитория

Один пользователь — автор продукта (личный инструмент). Возможное расширение: семья или команда.

## 2. Функциональные требования

### 2.1 Главный экран — «Сегодня»

| ID | Требование |
|----|------------|
| F-01 | Отображать текущую дату и день недели |
| F-02 | Показывать таймлайн дня с временными слотами (с 06:00 до 23:00) |
| F-03 | На таймлайне отображать события всех трёх блоков цветом по категории |
| F-04 | Кнопка «+ Добавить событие» открывает форму создания |
| F-05 | Индикатор текущего времени на таймлайне (красная линия) |
| F-06 | Счётчики в шапке: сколько событий сегодня по каждой категории |

### 2.2 Блок «Спорт» 🏃

| ID | Требование |
|----|------------|
| S-01 | Список запланированных тренировок на день и неделю |
| S-02 | Каждая тренировка содержит: тип активности, время начала, длительность, заметки |
| S-03 | Возможность отметить тренировку как «выполнена» |
| S-04 | Типы активностей: бег, тренажёрный зал, йога, плавание, велосипед, другое (ввод вручную) |
| S-05 | Отображение недельного прогресса (сколько тренировок выполнено из запланированных) |

### 2.3 Блок «Здоровье» 💊

| ID | Требование |
|----|------------|
| H-01 | **Приём таблеток:** название препарата, дозировка, время приёма, повторяемость (ежедневно / по дням недели) |
| H-02 | Возможность отметить таблетку как «принята» с фиксацией времени |
| H-03 | **Записи к врачу:** специальность врача, дата и время, адрес/клиника, заметки |
| H-04 | **Косметологические процедуры:** название процедуры, дата, мастер/салон, заметки |
| H-05 | Уведомление (визуальное на странице) о предстоящих событиях здоровья в ближайшие 24 часа |
| H-06 | История приёма таблеток за последние 7 дней |

### 2.4 Блок «Работа» 💼

| ID | Требование |
|----|------------|
| W-01 | **Встречи и совещания:** название, дата и время, участники (список имён), место / ссылка на конференцию, заметки |
| W-02 | **Задачи:** название задачи, дедлайн, приоритет (высокий / средний / низкий), статус (в работе / выполнена / отложена) |
| W-03 | Сортировка задач по приоритету и дедлайну |
| W-04 | Отметить встречу или задачу как выполненную |
| W-05 | Ближайшие встречи отображаются в блоке «Сегодня» |

### 2.5 Навигация и структура страниц

| Страница | Описание |
|----------|----------|
| `/` (Сегодня) | Главная: таймлайн + краткие сводки трёх блоков |
| `/sport` | Полный блок Спорт |
| `/health` | Полный блок Здоровье |
| `/work` | Полный блок Работа |
| `/week` | Вид на неделю — все события в сеточном календаре |

### 2.6 CRUD-операции

Для каждой категории событий:
- **Create** — форма добавления (модальное окно или inline-форма)
- **Read** — список / таймлайн
- **Update** — редактирование по клику на событие
- **Delete** — удаление с подтверждением

## 3. Нефункциональные требования

| ID | Требование |
|----|------------|
| NF-01 | Приложение работает полностью в браузере (может быть SPA без бэкенда на первом этапе) |
| NF-02 | Данные хранятся в `localStorage` (MVP) с возможностью последующего переноса на бэкенд |
| NF-03 | Адаптивная вёрстка: десктоп (≥1024px) и мобильный (≤768px) |
| NF-04 | Время загрузки первого экрана — не более 2 секунд |
| NF-05 | Работает в Chrome, Firefox, Safari последних двух версий |

## 4. UX / Дизайн

### 4.1 Тон и стиль

- **Дружелюбный**, тёплый, не строгий корпоративный
- Округлые карточки, мягкие тени, пастельные цвета по категориям
- Иконки (emoji или SVG-иконки) для визуальной навигации

### 4.2 Цветовая схема по категориям

| Категория | Основной цвет | Акцентный |
|-----------|--------------|-----------|
| Спорт | #4CAF82 (зелёный) | #E8F5EE |
| Здоровье | #6B9FD4 (голубой) | #EBF3FC |
| Работа | #F5A623 (оранжевый) | #FEF6E4 |

### 4.3 Ключевые UI-компоненты

1. **Таймлайн** — вертикальная лента с временными метками слева, события в виде карточек справа
2. **Блок-карточки** — три крупные карточки на главной, каждая ведёт в соответствующий раздел
3. **Форма добавления** — модальное окно с выбором категории в начале
4. **Бейдж статуса** — «✓ выполнено» / «⏰ скоро» / «⚠ просрочено»
5. **Прогресс-бар** — для спорта на неделе и приёма таблеток

## 5. Технический стек (рекомендации)

### Вариант A — чистый фронтенд (MVP, быстрый старт)

```
HTML + CSS (или Tailwind CSS)
Vanilla JavaScript (или Alpine.js для реактивности)
localStorage для данных
```

### Вариант B — React SPA

```
React + Vite
React Router v6 — навигация по страницам
Zustand или Context API — управление состоянием
Tailwind CSS — стилизация
localStorage через кастомный хук usePersistentStore
date-fns — работа с датами и временем
```

### Вариант C — с бэкендом (расширенный)

```
Frontend: React + Vite (как Вариант B)
Backend: Node.js + Express или Next.js (API Routes)
БД: SQLite (локально) или PostgreSQL
ORM: Prisma
Auth: опционально (JWT или NextAuth)
```

**Рекомендация для старта:** Вариант B — React SPA с localStorage. Это позволяет быстро разработать и при необходимости переключить хранилище на бэкенд.

## 6. Структура данных

### Событие (общая схема)

```typescript
interface Event {
  id: string;            // UUID
  category: 'sport' | 'health' | 'work';
  type: string;          // подтип (тренировка / таблетка / встреча и т.д.)
  title: string;
  date: string;          // ISO 8601: "2026-05-27"
  time: string;          // "09:00"
  duration?: number;     // в минутах
  notes?: string;
  status: 'pending' | 'done' | 'skipped';
  createdAt: string;
  updatedAt: string;
}
```

### Специфичные расширения

```typescript
// Приём таблеток
interface MedicationEvent extends Event {
  type: 'medication';
  medicationName: string;
  dosage: string;
  recurrence: 'daily' | 'weekly' | string[]; // ['mon','wed','fri']
  completedAt?: string;
}

// Встреча / совещание
interface MeetingEvent extends Event {
  type: 'meeting';
  participants: string[];
  location?: string;
  link?: string;
}

// Задача
interface TaskEvent extends Event {
  type: 'task';
  priority: 'high' | 'medium' | 'low';
  deadline: string;
}
```

## 7. Страница «Сегодня» — подробная логика

```
При загрузке:
1. Получить текущую дату
2. Загрузить все события из localStorage
3. Отфильтровать события на сегодня
4. Сгруппировать по категориям
5. Отрисовать таймлайн и блок-карточки

Таймлайн:
- Диапазон: 06:00–23:00 (68 слотов по 15 минут)
- Текущее время: красная горизонтальная линия + авто-скролл к ней
- Событие размещается на таймлайне согласно полям time + duration
- При клике на событие — всплывает попап с деталями и кнопками «Редактировать» / «Выполнено»
```

## 8. MVP vs Полная версия

### MVP (первая итерация)

- [ ] Главная страница «Сегодня» с таймлайном
- [ ] CRUD для всех трёх категорий
- [ ] Хранение в localStorage
- [ ] Адаптивная вёрстка
- [ ] Отметить событие как выполненное

### Полная версия (последующие итерации)

- [ ] Страница «Неделя» с сеточным видом
- [ ] Повторяющиеся события (напр., таблетки ежедневно)
- [ ] Push-уведомления в браузере (Web Notifications API)
- [ ] Экспорт в .ics (Calendar) / PDF
- [ ] Тёмная тема
- [ ] Синхронизация через бэкенд
- [ ] Статистика и аналитика (сколько тренировок за месяц, пропуски таблеток и т.д.)

## 9. Критерии приёмки (Definition of Done)

| Критерий | Описание |
|----------|----------|
| AC-01 | Пользователь может добавить событие в каждую из трёх категорий |
| AC-02 | События отображаются на таймлайне в правильном временном слоте |
| AC-03 | Пользователь может отметить любое событие как выполненное |
| AC-04 | Данные сохраняются между обновлениями страницы |
| AC-05 | Страница корректно отображается на мобильном устройстве (375px) |
| AC-06 | Удаление события требует подтверждения |
| AC-07 | Текущее время визуально выделено на таймлайне |
